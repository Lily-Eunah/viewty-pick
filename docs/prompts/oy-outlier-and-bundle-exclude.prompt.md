# Claude Code 작업 프롬프트 — 올리브영 매처 개선 (크로스셀러 가격 outlier 제외, 키워드 제외 없음)

> 목적: 올리브영(네이버 경유) 오퍼 선택 시, **디바이스/번들 등 이종 구성을 제외**하고, **다른 판매처/검색분포 가격대에서 동떨어진 outlier를 버리고 근접값을 채택**한다.
> 예) 바이오힐보 NAD: OY 후보 157,700(`[슈링크홈디바이스]…기획`=디바이스 번들) vs 39,000(`…세럼 30ml [단품/기획]`=단품) → 현재 "모호"로 둘 다 보류. 개선 후 **39,000 단품 자동 채택**.
> **올리브영(OY) 경로에만 적용**(운영자 지시). 네이버 브랜드스토어/쿠팡 앵커 로직은 건드리지 않음.
> 베이스: 최신 `main`. 분기 `feature/oy-outlier-bundle-guard`. 대상: `crawler/adapters/naver.ts`의 `pickOliveYoungOffer`/`matchOliveYoungOffer`.

## 변경 (⚠️ 키워드 기반 제외는 하지 않음 — outlier만)
> 운영자 지시: 디바이스/번들 키워드로 제외하지 말 것. 디바이스가 정품 구성인 경우도 있고 "2입 번들"처럼 범용이라 키워드 제외는 위험. **순수 가격 outlier로만** 거른다. (동종 1+1·증정 기획의 기존 gift-strip/개당가 처리는 유지.)

OY 후보가 여럿일 때 다음 순서로 단품을 고른다 (운영자 규칙):

1. **"단품" 문구 우선** — title에 `단품`이 있으면 그 후보를 우선(예: 바이오힐보 `…30ml [단품/기획]` 39,000). 증정/추가구성이 붙어도 "단품" 표기가 있으면 단품으로 본다.
2. **동일 제품명 후보 중 최저가** — 같은 제품(이름 일치)이 여러 개면 **가장 싼 것**을 채택(예: 몰바니 18,900 단품 vs 28,000 +50ml 증정기획 → 18,900).
3. **가격 outlier 제외** — 위로도 안 갈리면, 참조 가격대에서 동떨어진 후보 제거:
   - 참조 우선순위: ① 같은 product의 *다른 판매처(쿠팡/네이버) 매칭가* → ② 없으면 그 product의 **네이버 검색 결과 가격 분포 중앙값**.
   - 참조 대비 비정상(예: 중앙값의 2.5배 초과 또는 1/2.5 미만)인 후보 제외(예: 바이오힐보 157,700 디바이스 기획 → 28k~41k 분포의 outlier로 제외 → 39,000 채택).
4. 위 단계 후 후보 1개 남으면 채택. **여전히 모호해도 "가장 가까운 것"을 채택(last resort) + warning** — 운영자 검토용. *올리브영 후보가 ≥1개면 빈 채로(no price) 두지 말 것.* (올리브영 결과가 0건이면 link-only 유지.)
   - **키워드(디바이스/번들)로는 제외하지 않는다**(과제외 방지) — 디바이스/번들이 정품 구성일 수 있음.
   - "가장 가까운 것" 기준: 참조(타 판매처/검색분포 중앙값)에 가장 근접한 후보, 참조 없으면 동일명 최저가.
3. OY 신뢰밴드(sim≥0.6 + 코어토큰), 동종 1+1 개당가 처리 등 **기존 가드는 유지**. manual_override 우선순위도 유지.

## 테스트
- **바이오힐보 NAD**: 후보 157,700(디바이스 기획) + 39,000(단품) → 검색분포(28k~41k) 대비 **157,700을 가격 outlier로 제외** → **39,000 채택**. (키워드로 거르지 않음 — 가격만으로.)
- 키워드(디바이스/번들)로는 아무것도 제외하지 않는지 확인(과제외 방지).
- **몰바니 저자극 LHA**: 18,900(단품 200ml) vs 28,000(+50ml 증정 200ml 기획) = **같은 제품** → 동일명 최저가 규칙(2)으로 **18,900 채택**(hold 아님). 28,000은 증정 기획이라 gift-strip 후에도 더 비쌈.
- **hold/inspection은 *서로 다른 제품*으로 모호할 때만**(예: 토리든식 다른 SKU 근접 — 이름/정체가 갈리는 경우). *같은 제품의 가격 변형*은 hold 말고 최저가 채택.
- 네이버 브랜드스토어/쿠팡 경로 무변(회귀). `test:all`·typecheck·build·lint green. DB 쓰기 없음.

## 막히면
- 참조 가격 소스(다른 판매처 매칭가)가 매처 시점에 없으면 검색분포 중앙값으로 폴백. 그것도 빈약하면(후보=참조) 보수적으로 hold.
- outlier 임계(예: 중앙값 2.5배)는 보수적으로 시작하고, 정상 후보를 자르지 않게 테스트로 검증.
