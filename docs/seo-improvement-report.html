1현재 상태 진단 (실측)

1-1. 기술 SEO — 합격

잘 되어 있는 점 고유 title·canonical·OpenGraph, JSON-LD(ItemList·FAQPage·BreadcrumbList), 동적 sitemap, 모바일 우선, SSR(서버 렌더), 제품 4개 미만이면 404로 차단(빈 페이지 방지), 실시간 가격·제품 카드. → "최저가" 검색 의도에 맞는 실효용·신선도 신호가 있습니다.

1-2. 콘텐츠 — 얇고 전부 같은 틀 (가장 큰 문제)

2-2. 경쟁 현실: 신규 도메인은 헤드 키워드를 이길 수 없다

"선크림 추천", "토너 추천" 같은 헤드 키워드의 검색 결과 1페이지는 화해, 올리브영, 글로우픽, 네이버쇼핑 등 도메인 권위가 압도적인 사이트가 장악하고 있습니다. 신규 도메인이 이 벽을 넘으려면 수개월~수년의 권위 축적이 필요합니다.

전략적 결론 초기 유입의 유일한 현실 경로는 롱테일입니다. "민감성 약산성 토너 최저가", "PDRN 세럼 최저가 비교"처럼 3단어 이상·구체적·경쟁 낮은 키워드는 신규 사이트도 발행 후 비교적 빨리 상단에 노출되며, 전환율도 높습니다. 헤드 키워드는 롱테일로 권위를 쌓은 뒤 장기 타겟으로 둡니다.

2026 검색 트렌드는 성분·기능 중심의 정밀 검색(Fine Search)으로 세분화되고 있습니다(화해 F.I.N.D 리포트). 아래는 트렌드 키워드별로 ViewtyPick이 실제로 4개 이상 제품을 보유했는지를 카탈로그에서 측정한 결과 — 페이지로 만들 수 있는지의 근거입니다.

2.5-6. 실행 방안

  • 판매처 필터 신설 — lib/seo/match.ts에 seller 필터(해당 판매처 링크 보유 제품) 추가. seo_pages에 seller 컬럼 추가. 기존 구조의 자연스러운 확장.
  • 페이지 아키타입: "올리브영 {카테고리} 최저가 비교" — 올영 판매 제품 리스트 + 각 제품의 올영 가격 + "더 싼 곳" 배지(쿠팡/네이버가 N원 더 쌈). 올영 18개 카테고리 우선.
  • 핵심 본문 카피 — "올영에서 살 거면 이 토너 추천. 단, ○○는 네이버가 N원 더 쌉니다." → 정직한 비교 = 고유 콘텐츠 + 신뢰.
  • 사이트 피처: "지금 올영이 최저가인 제품" — 23개 제품 큐레이션(대분류 단위). "올영 충성 고객이 손해 안 보는 제품" 각도.
  • 세일 시즌 푸시 — 올영세일(3·6·9·12월) 검색 급증 타이밍에 "올영세일 최저가 체크" 콘텐츠·캐러셀 노출.
  • 검색량 검증 후 확정 — surffing/DataLab/Trends에서 "올리브영+카테고리+최저가/추천/인기" 조합의 검색량·포화도 확인 → "검색량 충분 + 포화 낮음 + 제품 4+" 조합만 페이지화(레티놀식 빈 페이지 방지).
카니발 주의 "토너 추천"(기존) vs "올리브영 토너 최저가"(신규)는 의도가 다르므로 공존 가능하지만, 제목·h1·도입부·canonical로 의도를 명확히 분리해야 합니다. 그리고 이 retailer 페이지들도 5장의 콘텐츠 깊이 규칙(고유 도입부·추천이유·업데이트일)을 그대로 적용해야 얇은 페이지를 피합니다.

2.6네이버 공략 — "웹페이지로 이기려 하지 말고 채널을 분리하라"

한국 화장품 검색의 본진은 네이버입니다. 그런데 네이버 통합검색에서 ViewtyPick 같은 웹 도메인이 들어갈 자리는 거의 없습니다. 이건 콘텐츠 품질 문제가 아니라 네이버 SERP 구조 자체의 문제라, "더 좋은 페이지"가 아니라 "다른 채널"로 풀어야 합니다.

2.6-2. 그래서 — 3채널 분리 전략

Play A — 네이버 블로그 채널 필수·최대 트래픽

공식 ViewtyPick 네이버 블로그를 운영해 "추천+최저가+성분" 롱테일 글을 발행하고, 본문에서 매칭되는 viewtypick.com 페이지로 링크합니다. 네이버 C-Rank는 주제 전문성 40%·지속성 30%를 보므로, 뷰티 한 주제에 꾸준히 쓰면 누적됩니다. ViewtyPick의 실시간 비교 데이터가 블로그 글에서도 다른 블로거가 못 가진 차별점이 됩니다.

Play B — 제품·브랜드 롱테일 웹 도메인·즉시 가능

"[브랜드][제품명] 최저가"는 블로그 포화가 낮고 거래의도가 강한 롱테일입니다. ViewtyPick은 이미 /p/[slug] 제품 상세 페이지에 "{브랜드} {제품명} 최저가 비교 · 매일 갱신" 메타를 갖췄습니다 — 93개 제품 = 93개 롱테일 타겟이 이미 준비돼 있습니다. 네이버에서도 웹사이트/스마트블록 영역, 구글에선 정면으로 노립니다.

Play C — 네이버쇼핑 가격비교 CP 제휴 장기·사업 검토

쇼핑 블록(네이버 검색의 최대 영역)에 들어가는 유일한 길은 네이버쇼핑 가격비교 CP로 등록하는 것입니다(다나와·에누리 방식). 사업 제휴가 필요해 난도는 높지만, 성공 시 "최저가" 검색의 본진을 차지합니다.

  • 본문에서 viewtypick.com 링크 — 블로그가 유입을 받아 웹으로 전환.
  • 테일윈드 — 2026 네이버는 AI 검색(Cue:)·스마트블록을 확대 중 → 구조화된 콘텐츠가 surface될 기회가 늘고 있음.

4왜 지금 구조로는 안 되는가 (원인)

원인 1 — 양산형 얇은/유사중복 페이지 → Helpful Content 필터

구글의 Helpful Content 시스템은 "검색엔진을 위해 만든 듯한" 페이지를 사이트 단위로 감지·강등합니다. 데이터 차별화가 없는 유사중복 페이지는 개별적으로 색인에서 걸러지고, 사이트의 얇은 콘텐츠 비율이 높으면 사이트 전체 평가까지 끌어내립니다. 2025년 12월 코어 업데이트에서 전문가 감수 없는 양산 콘텐츠는 87%가 부정적 영향을 받았습니다.

원인 2 — 신규 도메인의 권위 부족

도메인 권위가 낮은 신규 사이트는 경쟁 높은 헤드 키워드에서 대형 사이트의 벽을 넘지 못합니다. 권위는 단기간에 살 수 없고, 롱테일 유입 → 체류·재방문 → 점진적 권위 축적의 장기 루프로만 쌓입니다.

원인 3 — 고유 콘텐츠·경험(E-E-A-T) 부재

구글 품질평가와 네이버 D.I.A 모두 실제 경험·후기·전문성을 반영합니다. 현재 페이지엔 주제 설명 도입부, 작성자/전문성 표기, 제품별 추천 이유(데이터에 이미 존재하나 미노출), 가시적 업데이트일이 없어 "경험 신호"가 비어 있습니다.

원인 4 — 카니발리제이션으로 권위 자기분산

같은 제품군의 여러 페이지가 같은 키워드를 노리면 백링크·클릭·권위가 분산되어 어느 페이지도 임계점을 못 넘습니다. 하나로 모았다면 상위 노출됐을 권위가 쪼개집니다.

원인 5 — FAQ 리치 결과 폐지 (2026.5.7)

구글은 FAQ 리치 결과를 2026년 5월 7일부로 일반 사이트에서 폐지했습니다(정부·의료 등 권위 사이트만 노출). 현재 40개 동일 FAQ 스키마는 SERP 리치 결과를 만들지 못합니다. 다만 FAQPage 스키마는 AI Overview/생성형 인용에는 3.2배 유리하므로, 제거가 아니라 "페이지마다 고유한, 진짜 유용한 FAQ"로 전환해야 가치가 있습니다.

5개선 방안 (우선순위별)

P0 카니발리제이션 정리

  • 완전 동일 쌍 통합 — acne-pad·men-allinone은 (a) 약한 쪽을 강한 쪽으로 canonical 처리하거나, (b) 정렬·필터·카피를 실제로 다르게 만들어 차별화. 차별화가 불가능하면 비활성(404)이 정답.
  • ≥0.8 쌍 차별화 — device-best/dry-device, cleansing-oil-best/blackhead-cleansing-oil 등은 "건성 전용 사용법", "블랙헤드 집중" 등 고유 관점·정렬·콘텐츠를 부여.
  • 대분류=허브 역할 분리 — skincare/cleansing/maskpack 페이지는 제품 나열이 아니라 자식 페이지(토너/세럼/크림…)를 링크하는 허브로 재정의 + 고유 가이드 카피. 검색의도가 다르므로 유지하되 역할을 분리.
원인 같은 제품 페이지가 서로 순위를 깎고 색인 예산을 낭비하기 때문. 통합하면 분산됐던 권위가 한 페이지에 모입니다.

P1 페이지 콘텐츠 깊이 — 랭킹의 최대 레버

  • 페이지별 고유 도입부 2–4문장 — 주제 설명 + 선정 기준 + 구매 팁. 카테고리·피부타입·키워드를 자연스럽게 포함. (300–500자만 더해도 효과 큼)
  • 제품별 "추천 이유" 노출 — reasonItems가 데이터에 이미 있는데 랜딩에서 안 보입니다. 카드에 1–2줄 노출 → 고유 콘텐츠 + 경험(E-E-A-T) 동시 충족.
  • 주제별 고유 FAQ 1–2개 — "여드름 패드 매일 써도 되나요?"처럼 페이지 주제에 맞는 질문으로 교체(공통 3개 → 일부만 공통 + 주제별 고유).
  • "최종 업데이트 N월 N일" 노출 — 매일 가격을 크롤하므로 신선도를 가시화(lastUpdated 데이터 활용). 신선도는 구글·네이버 공통 신호.
  • 고유 메타 설명 — 공통 접미사("…한 번에 비교했어요") 제거 또는 변주. 최저가/제품수/대표 브랜드를 넣어 차별화 + CTR↑.
  • 최저가 TOP3 요약 블록 — 상단에 "현재 최저가 3" 표/카드로 즉시 가치 전달(고유 데이터 + 거래의도 충족).
원인 고유 콘텐츠·경험 신호가 비어 양산형으로 평가되기 때문. 같은 데이터라도 "고유하게 가공"하면 Helpful Content 통과 + AI Overview 인용 가능성↑.

P2 롱테일·성분 키워드 전략

  • 이길 수 있는 키워드만 타겟 — 신규 도메인은 헤드("토너 추천") 대신 롱테일("민감성 약산성 토너 최저가"). 제목·h1·도입부에 롱테일을 자연 포함.
  • 트렌드 성분 페이지 신설 — PDRN(5개)·시카 진정(30개)은 수요 급증 + 제품 충분 → 즉시 페이지화. 약산성/판테놀/톤업(각 4개)은 고유 콘텐츠 전제로 추가.
  • "제품 4+ · 검색량 충분 · 포화도 낮음" 3조건을 신규 페이지 추가 규칙으로 코드화(lib/seo/specs.ts). 제품 0개 키워드(레티놀·엑소좀)는 금지.
  • 제목 패턴 다양화 + '2026' 동적화 — 전부 "X 추천 최저가 비교 2026"는 차별성↓·연도 노후화. 연도는 빌드시 자동 치환, 일부는 의도/혜택형 제목으로 변주.

P4 기술·구조·AI 최적화

  • 내부 링크 강화 — 각 페이지 하단에 "관련 가이드"(같은 카테고리·피부타입·성분) 3–6개 맥락 링크. 허브↔자식↔형제 링크로 권위 순환·카니발 신호 정리.
  • FAQPage 스키마 유지하되 고유화 — SERP 리치 결과는 끝났지만 AI Overview 인용에 유리. 단, "페이지마다 다른 진짜 FAQ"일 때만.
  • ItemList·Product·Offer 스키마 강화 — 가격·통화·재고·평점(있으면)을 정확히. 비교 사이트의 핵심 신호.
  • AI 검색(GEO) 대비 — 명확한 소제목·표·요약 블록으로 "AI가 추출하기 쉬운 구조". 2026 검색은 "AI가 먼저 읽는다".

6캐러셀 · 인덱스 페이지 품질 개선

6-1. 홈 추천 캐러셀

현재 5개 카드(선크림·기초 스킨케어·클렌징·쿠션·디바이스)는 제품 수 많고 주제가 뚜렷한 상위권이라 선택 자체는 적절합니다. 다만 "검색 유입 → 클릭 → 체류"를 끌어올릴 여지가 있습니다.

  • 구체 수치·후킹 — 카드에 "추천 N개 · 최저가 ₩X~", "오늘 기준" 등 실데이터 노출(신뢰·CTR↑).
  • 실제 제품 썸네일 — 이모지 대신 대표 제품 이미지 2–3개 미리보기(체류·기대 일치).
  • 검색의도 키워드 포함 카피 — "민감성 선크림 최저가"처럼 카드 문구·alt에 타겟 키워드 반영.
  • 클릭 데이터 기반 선순환 — 캐러셀·허브 클릭을 트래킹해 인기 가이드를 자동 노출(이미 affiliate_clicks 인프라 존재). 인기↑ 페이지에 내부링크 집중 → 권위 가속.
  • 비활성/얇은 페이지는 승격 금지 — 4개짜리·중복 페이지는 캐러셀에 올리지 않음(품질 신호 보호).

6-2. /best 인덱스(허브) 페이지

현재는 그룹별 텍스트 링크 목록입니다. 허브도 얇으면 권위 전달·자체 랭킹이 약합니다.

  • 가이드 카드화 — 각 가이드에 대표 이미지 + 제품 수 + 최저가 미리보기 + 1줄 설명(고유).
  • 그룹별 고유 인트로 — "피부 타입별", "성분·고민별" 등 각 섹션에 2–3문장 고유 설명(허브 자체의 콘텐츠 깊이).
  • 정렬·필터 — 인기순/신규순/카테고리 필터로 탐색성↑ + 체류↑.
  • Breadcrumb·내부링크 허브화 — 허브가 모든 /best 페이지로 링크되는 구조 → 색인 발견성·권위 분배 최적화.
  • 허브 자체 SEO 타겟 — "뷰티 추천 가이드 모음", "화장품 최저가 비교" 등 미들 키워드를 허브가 흡수.

7실행 로드맵

1
1단계 — 정리 (런치 전 필수) · P0 카니발 통합/차별화, 비활성 정리, canonical 정비. → 색인할 페이지를 "이길 수 있는 것"만 남김.
2
2단계 — 콘텐츠 깊이 · P1 전부(고유 도입부·추천이유 노출·주제별 FAQ·업데이트일·고유 메타·TOP3 블록). → 양산형 탈출.
3
3단계 — 키워드 확장 + 발견성 · 판매처×최저가 아키타입(2.5): "올리브영 {카테고리} 최저가 비교" 페이지(올영 18개 우선) + seller 필터 구현. P2 롱테일/트렌드(PDRN·시카) 페이지, P4 내부링크·스키마, 캐러셀·허브 품질(P3). 그 후 SITE_INDEXABLE=true + GSC/네이버 등록 + sitemap 제출.
4
4단계 — 네이버·외부 권위(지속) · P5 네이버 웹마스터도구, 데이터랩 기반 키워드 갱신, 블로그/인플루언서 교차 노출로 C-Rank 주제 권위·백링크 축적. 클릭 데이터로 캐러셀 선순환.
