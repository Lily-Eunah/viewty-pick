## 0. 출발점 — 무엇이 문제였나

기존 `/best`(SEO 추천 가이드 허브, ~50개 링크)를 실제로 열어보고 나온 진단:

- **시각 계층이 없다**: 46개 가이드가 전부 동일한 흰 카드 텍스트 리스트. 아이콘·이미지·배지 없음
- **"N개 →"가 유일한 클릭 신호**: 제품 수인지 안내인지 불분명, 클릭 유도력 약함
- **그룹 간 차별화 없음**: 카테고리별/피부타입별/고민·성분별/큐레이션이 배경·타이포 전부 동일
- **Featured 개념 없음**: 46개가 전부 평등하게 나열, 진입점이 없음
- **OliveYoung 전용 페이지가 안 보임**: 시각적으로 일반 카드와 동일

## 1. 경쟁사 리서치 → 디자인 방향 확정

올리브영·화해·무신사뷰티·Sephora·Glossier·ASOS 6곳의 큐레이션/허브 페이지 UX를 리서치. 채택한 패턴:

| 레퍼런스 | 채택한 것 |
|---|---|
| 무신사 뷰티 | 섹션별 컬러 코딩(올리브영 밴드를 초록으로 분리) |
| 화해 | 카테고리 아이콘으로 즉각 스캔어빌리티 |
| Sephora | Featured 히어로 + 실제 상품 스포트라이트 |
| 올리브영 | 가격/할인 배지를 시각 최우선으로 |

## 2. 정보구조 재설계 — "taxonomy가 아니라 차별적 가치 기준"

사용자 피드백("선크림 최저가는 카테고리 탭에서 봐도 되는 거 아냐?")을 계기로 그룹핑 원칙을 바꿈:

> **"카테고리 탭으로 못 얻는 차별적 가치 × 검색 의도"** 로 재정렬. taxonomy(페이지 종류)가 아니라
> "이 페이지가 카테고리 필터와 뭐가 다른가"를 기준으로 tier를 매김.

결과 4-tier 순서:
1. **에디터 픽** (curation) — 필터로 못 만드는 전문가 큐레이션
2. **피부 타입별** — 카테고리 필터로도 되지만 검색 의도가 다름("건성 추천")
3. **고민·성분별** — 카테고리를 가로지르는 슬라이스
4. **올리브영** — 판매처 슬라이스, 별도 밴드로 분리
5. **카테고리별** — 카테고리 탭과 완전 중복 → **하단 chip으로 강등** (삭제는 안 함: SEO head-term 유입 자산)

## 3. 아이콘 전략 — "1 + 2(b)"

- 카테고리 축: 기존 디자이너가 만든 `public/images/categories/*.png` 대분류 일러스트 재사용
- 고민/성분·피부타입 축: 대응하는 기성 일러스트가 없어서 두 선택지 검토
  - (a) 원 디자이너에게 스타일 확장 의뢰 — 최선이지만 리소스 필요
  - (b) 직접 mauve 라인 SVG 세트 제작 — 무료·즉시
  - → **(b)로 진행**, `components/seo/GuideIcon.tsx` (제품형 12종 + 개념형 6종)
- 이모지 검토: 프로덕션 UI엔 **비추천** 결론 (플랫폼별 렌더 불일치, 브랜드 팔레트와 충돌, "믿고 사는 최저가" 포지션과 안 맞음) — 단, 에디터 픽 히어로처럼 "확 튀어야 하는" 자리엔 Twemoji(CC-BY) 채택

## 4. 성분/피부 "전체" 허브 — 검색 결과와의 괴리 발견

**계기**: "PDRN을 검색하면 5개 나오는데 왜 /best엔 없냐"는 질문.

**원인 분석**: SEO 페이지 매칭(`matchSeoProducts`)은 `category` **AND** `keywords`인데, 검색(`matchesQuery`)은 **키워드 단일 조건**. `pdrn-serum`(category=serum AND keyword=PDRN)은 세럼인 PDRN만 잡아 2개 → 4개 미달로 비활성. 실제론 PDRN이 토너·크림·세럼·선크림·클렌징에 총 7개 분산.

**해결**: 카테고리 조건 없는 "전체" 허브 신설 (`soothing`/`hydra`/`pdrn`/`acne`/`blackhead` + `dry-skin`/`oily-skin`/`sensitive-skin`/`combination-skin`/`dehydrated-skin`, 총 10개). 드릴다운 타일을 탭하면 `[전체] + 카테고리별 개별 페이지`가 함께 펼쳐지는 구조로 통일.

## 5. 드릴다운 UX 반복 — 4번의 피드백 라운드

1. **아코디언 초안**: 탭하면 세로로 펼침, 첫 그룹 기본 펼침
2. → "첫 그룹도 접어줘" — 전부 접힌 상태가 기본으로
3. → "정사각형 이모지 타일 + 가로 나열(2줄)로" — 올리브영 밴드 스타일과 통일
4. → "이모지와 라벨이 너무 붙어있다" — 타일 내부 gap 조정 (`gap-1` → `gap-2`)
5. → "고민·성분 재구성" — 진정/수분/PDRN(성분 중심)과 여드름/블랙헤드(피부 타입 섹션으로 이동) 분리

## 6. 에디터 픽 — "재미없다"에서 "딜 카드"로

**1차 진단**: 히어로 1개 + 세로 리스트 3줄 — 매일 같은 픽, 가격 비교 서비스의 강점(가격 훅)을 안 씀.

**디자인 검토 과정**:
- 멀티카드 레일(244px) 제안 → "가장 좋은 안인지 재검토해달라" 요청에 자체 비판
  → 카드가 작아 가격 드라마(큰 숫자·취소선·%↓)가 안 산다는 결론
- **최종안**: 풀폭(300px) 페이지드 캐러셀 + CSS `scroll-snap` + 다음 카드 peek + 진행 dots
  (JS는 dots 추적만, 링크는 전부 서버 렌더 DOM에 상주 → SEO 보존)

**후킹 카피 전략**: "어그로를 끌되 신뢰를 깨지 않는" 원칙 —
- 시의성 자극 ("UV지수 최악의 계절")
- 실데이터 할인율 (%↓는 실제 `정가` 대비 계산값)
- "최저 판매처" 배지 (쿠팡/네이버/올리브영 색상 dot)
- 네이버 트렌드 기반 자동 성분 추천은 **TBD로 보류** (지금 스코프 아님)

**버그 2건 발견·수정**:
1. **시즌 4장 중 선크림 3장 + 완전 동일 카드 2장** — 원인: (a) 시즌 구성이 선크림 위주로 편중 (b) 톤업↔디렉터파이 가이드가 제품 풀을 공유해 최저가 상품이 수렴
   → 구성 다양화(선크림/진정/수분/PDRN으로 각도 분산) + 코드 dedupe(카드마다 이미 쓴 상품·브랜드를 추적해 다음 후보로 스포트라이트)
2. **카드 대표 상품이 가이드 페이지의 실제 1등과 다름** — 원인: 스포트라이트 후보를 최저가순으로 재정렬했는데, 가이드 페이지 자체는 recommend(ViewtyScore) 순으로 보여줌
   → **recommend 순서 그대로 사용**하도록 통일 (최저가 배지 값만 별도로 진짜 최저가 유지)
