# feature/product-card-seller-logos — 리스트 카드: 판매처 로고 + 할인율 가격 옆 재배치

- **일자**: 2026-07-17
- **목표**: 카테고리 리스트 카드에서 (1) `정가 대비 n%↓` 뱃지가 `디렉터파이 추천` 뱃지와 줄바꿈되는 문제, (2) `올리브영 · 쿠팡 · 네이버 비교` 텍스트가 iOS에서 줄바꿈되는 문제 해결.

## 원인 (375px DOM 실측)

- 앱 컨테이너 max-width가 **430px** → 375px 기기에서 카드가 328px로 축소, 뱃지 행 가용 폭 **118px**.
- 뱃지 2개가 필요로 하는 폭 **146px**(`디렉터파이 추천` 87px + 할인 뱃지 53px + gap 6px) → **28px 초과, 37/37 카드 전부 줄바꿈**.
- 430px 이상(iPhone Pro Max·데스크톱)에선 여유 +42px로 **줄바꿈 0** → 좁은 기기(SE/mini)에서만 발생하는 문제였음.
- `정가 대비` 문구 제거만으로는 45px밖에 못 벌어 **여전히 12px 부족**했음.
- **진짜 원인은 `pr-10`(40px)**: 찜 버튼은 카드 우상단에 `absolute`로 떠 있는데 여백이 오른쪽 영역 **전체**에 걸려 있어, 하단 로고까지 안쪽으로 밀리고 하단 40px이 통째로 놀고 있었음.

## 결정

1. **판매처 텍스트 → 실제 브랜드 앱 아이콘**: 네이버(초록 N) / 쿠팡(톱니 씰) / 올리브영(O+올리브)을 64px PNG로 `public/images/sellers/`에 저장. 폭 20px 고정이라 줄바꿈 원천 차단. 로고 없는 판매처(zigzag/ably)는 텍스트 폴백 없이 **건너뜀**.
2. **할인율을 뱃지 행 → 가격 왼쪽으로 이동**: 할인율은 제품 속성이 아니라 **가격 정보**. `61%↓ 24,900원`이 한 덩어리로 읽히고, **할인율순 정렬 시 왼쪽 열에 세로 정렬돼 스캔** 가능(정렬 기준은 그 화면에서 가장 잘 읽혀야 함). 커머스 표준 패턴(쿠팡·네이버·무신사).
3. **뱃지 행은 제품 속성 전용**으로 비움 → 향후 화해·올영 랭킹 뱃지 추가 여지 확보(31px 여유).
4. **`pr-10`을 상단 블록에만 적용** → 하단 40px 회수, 로고가 찜 버튼과 무관하게 카드 최우측 정렬.

**기각안**
- *이미지 위 할인율 오버레이*: 제품 사진 9% 가림 + 순위 뱃지와 좌측 충돌. 뷰티는 패키지 자체가 상품 정보라 사진 침범 비용이 큼.
- *`최저가 61%↓` 라벨 줄 병기*: 라벨(9px 회색)과 값(11px 버건디)의 위계가 모호하고, 정렬 기준이 되기엔 너무 작고 흐림.
