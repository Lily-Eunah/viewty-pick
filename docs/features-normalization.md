# Features 요약·정규화 (products 탭)

제품 추천 이유로 화면에 노출하기 위해 `features` 컬럼을 정규화한 작업 결과.

## 규칙
- **문체**: 쉼표로 구분된 짧은 명사구 4~6개. `prod.features.split(',')` → 칩 렌더링 + description으로 사용.
- **구성 순서**: **정체성 구문(제형/규격) 먼저** → 핵심 성분·기능 → 효과·사용감 → 추천 피부타입 → 인기·세그먼트 태그.
  - **첫 항목 = 정체성 구문**: "OO 토너 / OO 선크림 / OO 앰플"처럼 제품이 무엇인지 한눈에 드러나는 명사구로 시작한다. 화면(RecommendationReasonBox)에서 이 첫 항목이 리드 라인으로 강조되므로, `SPF50+ PA++++` 같은 스펙만 앞에 오지 않게 한다.
- **인기·세그먼트 태그 유지**: `올영 인기`, `남성 인기`, `남성 추천`, `백화점 Top`, `더마 브랜드 TOP`, `온라인 브랜드 TOP`, `온가족 사용` 등은 그대로 보존. (신규 태그는 운영자 큐레이션 영역 — 웹 리서치로 임의 추가하지 않는다.)
- **긍정 추천 이유만**: `민감 피부 주의`, `지성 비추천`, `호불호` 등 주의·비추천 표현은 요약에서 제외 (원문 detail 컬럼에 보존).
- **원본 보존**: 직접 작성한 상세 내용은 별도 컬럼(`features_detail`)에 그대로 저장.
