# 제목 용량/구성 파싱 규칙 Spec (현행 + 개선안)

> **이 문서는 spec/참고용이며 `feature/volume-unit-and-price-display` branch에는 커밋하지 않는다.**
> 후속 branch(`fix/title-volume-parsing` 제안)에서 다룰 내용을 정리한다.
> 관련 코드: `crawler/core/packageExtractor.ts`, `crawler/core/normalize.ts`, `crawler/adapters/naver.ts`.

---

## 1. 목적

판매처(네이버/쿠팡/올리브영) 제품 제목에서 **용량/수량/구성**을 정확히 읽어
ml당(또는 매당/g당) 단가를 계산한다. 핵심 원칙:

- 제목에 **용량 표기(ml/매/g)가 없으면** → 운영자 입력 `volume_ml`을 그대로 사용.
- 제목에 **용량 표기가 있으면** → 파싱해서 사용(판매처별 용량이 다를 수 있음).
- **증정/사은품/샘플**은 본품 용량·수량에 포함하지 않는다.
- **본품의 추가 구성(리필/동일제품 +N)** 은 수량/용량에 포함한다.

---

## 2. 현행 동작

### 2-1. 용량 결정 순서 (`normalize.ts:182-200`)
1. 어댑터 `parsedVolumeRaw` (네이버 매칭 제목에서 파싱한 ml)
2. 없으면 `extractPackageFromTitle(제목)` — 단 `unitAmount !== null` 일 때만 적용
3. 둘 다 없으면 **운영자 `product.volume_ml`** 로 폴백

### 2-2. 파싱 규칙 우선순위 (`packageExtractor.ts`, 위에서부터 먼저 매칭)
| # | 패턴 | 예 | 결과 |
|---|------|-----|------|
| 1 | additive `A+B`(ml/g) | `50ml+50ml` | 동일→번들 ×2 합산 / 배수+개수신호→동종멀티팩 / 그 외→**heterogeneous(제외)** |
| 1b | 리필·본품+ | `265ml 본품+리필` | 동일제품 ×2 |
| 1c | `N+N` | `1+1`, `2+1` | x+y개 |
| 1d | 더블기획/팩/구성 | `더블기획` | ×2 (단, `대용량` 제외) |
| 2 | 시트+용량 | `25ml, 10매입` | sheet, 매수=10 |
| 3 | 용량×N | `60ml x 2` | ×2 |
| 4 | 용량+개수 | `60ml 2개입` | ×2 |
| 5 | 시트(매)만 | `패드 70매` | sheet, 매수=70, **`unitAmount=null`** |
| 5b | heterogeneous | `토너100ml+세럼30ml`, 세트/패키지/디바이스 | 제외+검수 |
| 6 | 단일 용량 | `60ml` | 단일 |

### 2-3. 증정/구성 분리 (`stripPromoGifts`)
- **제거(증정):** `+…증정/사은품/샘플`, `미니/트래블/여행용/파우치/키트/쇼핑백/덤/마스크`, 괄호 내 증정류.
- **유지(본품 구성):** `리필`, `본품+`, 동일용량 `A+A`.

### 2-4. heterogeneous 복구 (네이버 한정, `naver.ts:363-489`)
앵커 매칭된 SKU가 heterogeneous로 보여도, **소량 부스트/add-on**을 떼고
**본품(=DB volume)** 에 가격을 귀속(보수적). 쿠팡/올영에는 이 복구 없음.

## 3. 식별된 갭 (개선 대상)

| # | 갭 | 현재 | 기대 |
|---|----|------|------|
| G1 | **매(sheet) 파싱이 normalize까지 일관되지 않음** | case 5가 `unitAmount=null` → `normalize.ts:194` 조건에서 volume 미적용. 매 용량은 어댑터 `parsedVolumeRaw`로만 유입 | 매 제품도 제목의 매수를 일관되게 volume으로 사용 |
| G2 | **동일제품 비배수 additive 합산 미지원** | `75ml+35ml`(같은 제품) → heterogeneous 제외 | `더하기`로 총 110ml 단일 취급(같은 제품일 때) |
| G3 | **다른제품 증정 간주가 네이버 한정·휴리스틱** | `토너75ml+세럼35ml` → 네이버만 본품 귀속, 쿠팡/올영 제외 | 제품명 기반으로 본품 식별 → 다른 제품은 증정 처리, 본품 용량 사용 (전 어댑터 공통) |

### 개선 방향 (후속 branch)
- **G1:** case 5(sheet)에서 `unitAmount`에 매수를 싣거나, `normalize.ts`가 sheet 단위를 별도 처리. 단위 인프라(`volume_unit`)와 연동해 매당 계산 일관화.
- **G2:** additive `A+B`가 **같은 제품**(주변 토큰에 제품명/카테고리어 1개)일 때 합산, **다른 제품명**이면 heterogeneous 유지. DB 제품명/카테고리와 대조해 판단.
- **G3:** `+` 양쪽 토큰을 제품명으로 분리 → DB 매칭 제품과 일치하는 쪽을 **본품**, 불일치하는 쪽을 **증정**으로 strip. 네이버 전용 로직(`naver.ts`)을 공통 유틸로 승격.

### 주의
- 용량/단가 수치를 바꾸므로 **전 제품 회귀 위험**. `packageExtractor.test`·`normalize.test`·어댑터 테스트로 충분히 커버 후 진행.
- 단위 인프라(`volume_unit`)가 먼저 머지된 뒤 진행(매당 일관 계산 전제).

## 4. 예시 (기대 결과)
| 제목 | 현행 | 개선 후(기대) |
|------|------|---------------|
| `토너 70매` (DB 70매) | 매수 70 (어댑터 의존) | 매수 70 일관 |
| `패드` (용량표기 없음, DB 70매) | 운영자 70매 사용 ✅ | 동일 |
| `세럼 50ml + 50ml` | ×2 합산 ✅ | 동일 |
| `세럼 75ml + 35ml` (동일제품) | heterogeneous 제외 ❌ | 110ml 합산 |
| `토너 75ml + 세럼 35ml 증정` | 증정 strip ✅(증정어 있을 때) | 동일 |
| `토너 75ml + 세럼 35ml` (증정어 없음) | 네이버만 본품 귀속 | 전 어댑터 본품(토너75) 사용, 세럼 증정 |
