## 5. 교차 근본원인 & 조치
1. **큐레이트 URL이 앵커 불가** — 네이버 7건이 비-`/products/{N}` URL, 넘버즈인 쿠팡은 short-link. → **운영자: 제품 상세 URL로 교체** (네이버 `brand.naver.com/{store}/products/{N}`, 쿠팡 `/vp/products/{N}`). 가장 큰 회수 효과.
2. **브랜드 오타(이지앤트리→이즈앤트리)** — 시트 수정됨, 재import 해결.
3. **recall 깊이** — 앵커가 검색 상위(display=5/10)에 들어야 매칭. display 수를 늘리거나 link-resolution(item.link→productId)을 보강하면 일부 앵커 miss 회수 가능.
4. **OY 신뢰밴드 보류(몰바니·바이오힐보)** — 수동 검토 → 맞으면 manual_override.
5. **랑콤 등 백화점 brand** — 쿠팡/OY 미취급 → 네이버 공식몰만으로 정상(고칠 것 없음).

## 6. #2 공식가(정가) — 할인 전 가격 확보 방법

**사실 확인:** 네이버 쇼핑 검색 API 응답 필드 = `lprice`(현재가=할인가), `hprice`(빈 값), `mallName`, `productId`, `productType`, `brand`, `category…`. **정가/할인전가 필드 없음.** → DB `regular_price` 40/40 null → 공식몰 대비 픽의 "공식가"가 할인가로 잡힘.

| 방법 | 가능? | 비고 |
|---|---|---|
| 네이버 쇼핑 검색 API | ❌ | 정가 필드 자체가 없음 |
| 브랜드몰 상세페이지 크롤 | △ | 스마트스토어/brand.naver.com 페이지엔 `salePrice`(정가)+`discountedSalePrice`(할인가)가 내장. 매처가 이미 channelProductNo를 알고 있어 URL 확보됨. **단 anti-bot·JS 렌더·취약**(API-first 위배), best-effort만 |
| **운영자 정가 컬럼 (권장)** | ✅ | products(또는 product_links)에 `정가` 컬럼 추가 → 운영자가 공식 정가 입력 → 크롤러가 `regular_price`로 사용. **정가는 거의 안 변해 유지보수 적고 100% 정확**, 시트 주도 모델과 일치 |
| 메트릭 재정의 | ✅ | "공식 스토어 **현재가** 대비 최저가"로 프레이밍 변경(정가 불필요) — 단 체감 할인폭은 약해짐 |

**권장:** 정가 컬럼(운영자 입력) + 크롤러가 그것을 regular_price baseline으로 사용. 비어있으면 지금처럼 sale_price 폴백.

## 7. 제안 액션
- **(즉시·재크롤)** `crawler:sync` 1회 → warning 잔상 정리 + 이즈앤트리 오타 반영(검색 0→정상).
- **(운영자 데이터)** 앵커 불가 URL 교체: 네이버 7건 비-products URL, 넘버즈인 쿠팡 short-link, 에스트라 쿠팡 productId 확인.
- **(검토)** 몰바니·바이오힐보 OY 보류 수동 확인 → 맞으면 override.
- **(#2)** 정가 컬럼 도입 여부 결정 → 결정 시 시트 컬럼 + 크롤러 regular_price + 공식몰대비 메트릭(정가 우선·없으면 sale 폴백) 반영.
