# 정가·용량 단위 처리 기준 및 운용 가이드

> 대상 branch: `feature/volume-unit-and-price-display` (구현 예정)
> 이 문서는 **정가(regular_price) 대비 할인** 표시와 **용량 단위(ml/매/g)** 처리의
> 기준, 구현 후 바뀌는 **운영자 시트 입력 방법**, 화면 표시 규칙을 한 곳에 정리한다.

---

## 1. 배경 / 목적

- 할인율은 **시트에 입력한 정가(regular_price)** 기준으로만 표시한다. (공식몰 폴백 제거 완료)
- 그런데 제품 용량이 항상 ml이 아니다 — **패드는 매(장), 일부 제품은 g**. 현재는 모든
  용량을 `ml`로 하드코딩 표시해 패드/그램 제품이 어색하게 보인다.
- 본 작업으로 **단위(ml/매/g)를 제품별로 관리**하고, 정가·최저가·단위당가 표시를
  단위에 맞게 정리한다. **할인율 계산 로직 자체는 바꾸지 않는다.**

---

## 2. 핵심 처리 기준 (가장 중요한 규칙)

### 2-1. 정가(regular_price)
- **출처:** 운영자가 `products` 시트의 `regular_price` 칸에 입력한 값 = 그 제품의 **정가/MSRP**.
- **단위 기준:** 정가는 항상 **DB 대표 용량(`volume_ml`) 1개 기준**의 가격이다.
  (예: 70매 패드의 정가 = 70매 1팩의 정가)
- **입력 형식:** `₩35,000`, `35,000원`, `35000` 모두 허용 — 숫자만 추출해 저장한다.
  (₩·콤마·원·공백 제거 후 파싱. 0/음수/빈칸 → 정가 없음으로 처리)
- **정가가 없으면:** 할인 배지·정가 표시 없음, "정가 대비 최저가 픽"에서도 제외.
  (오표시 방지 — 억지로 공식몰 등 다른 기준을 끌어오지 않는다)

### 2-2. 용량(volume_ml) 과 단위(volume_unit)
- `volume_ml` = **용량 숫자**(컬럼명은 ml이지만 실제로는 "수량 값").
- `volume_unit` = 그 숫자의 **단위** — `ml` · `g` · `매` 중 하나. **기본값 `ml`**.
  - 시트에 `장`/`시트`/`p`를 적으면 `매`로 정규화. 그 외 알 수 없는 값은 `ml`로 폴백.
- ml 제품은 **지금과 100% 동일**하게 보인다 ("70ml", "ml당 198원"). 운영자는
  **패드(매)·그램(g) 제품만** `volume_unit`을 채우면 된다.

### 2-3. 할인율 계산 (= 현행 유지, "단위당" 정규화)
할인율은 **단위당(ml당/매당/g당) 가격**으로 계산한다. 판매처마다 용량이 다를 수 있어
총가가 아닌 **단위당가**로 비교한다(작은 용량이 싸 보이는 착시 방지).

```
정가 단위당가 = regular_price / volume_ml
판매처 단위당가 = 판매가 / (그 판매처의 용량 × 수량)
할인율(%)     = round((정가 단위당가 − 판매처 단위당가) / 정가 단위당가 × 100)
```

- 판매가 ≥ 정가(낡은 정가) → **0%로 clamp** (음수 할인 없음).
- 정가/용량/판매처 단위당가 중 하나라도 없으면 → 할인 **미표시**(null).

#### ⚠️ 용량이 다른 판매처가 최저가가 되는 경우 (예: 비플레인 시카테롤 블레미쉬 패드)
| 항목 | 값 |
|------|-----|
| DB 대표 용량 / 정가 | **70매 / 26,000원** → 정가 매당 약 **371원** |
| 최저가 판매처 | **185매 / 14,250원** → 매당 약 **77원** |
| 할인율 | (371 − 77) / 371 ≈ **79%** |

→ 79%는 **185매 대용량 팩의 매당 가격**을 70매 정가의 매당 가격과 비교한 결과로,
계산상 정상이다. 다만 사용자가 "정가 26,000 → 최저가 14,250"만 보면 79%가 커 보이므로,
**아래 표시 규칙(3장)으로 혼동을 줄인다.**

## 3. 화면 표시 규칙 (구현 후)

### 3-1. 홈 "정가 대비 최저가 픽" 카드 (`TodayDealSection`)
- **정가 금액은 표시하지 않는다.** (정가 vs 최저가 숫자가 79%와 어긋나 보이는 혼동 제거)
- 할인 배지 `정가 대비 79% 할인`은 **유지**.
- **단위당가**(예: `매당 77원`)를 **할인가(최저가) 우측**으로 이동.
- 예시 레이아웃:
  ```
  [정가 대비 79% 할인]
  14,250원   매당 77원
  쿠팡 · 올리브영 비교
  ```

### 3-2. 제품 리스트/그리드 카드 (`ProductListCard`, `ProductCard`)
- 변동 없음(원래 정가 금액 미표시). 할인 배지 `정가 대비 N%↓` 유지.

### 3-3. 상품 상세 페이지 (`app/p/[slug]`)
- **용량/규격 줄에 정가를 함께 표기**(있을 때만):
  `용량/규격: 70매 · 정가 26,000원`
  → 정가가 "이 용량/규격 기준 가격"임을 명확히 한다.
- 가격 영역의 **최저가 옆 취소선 정가는 제거**, 할인 배지(`정가 대비 79% 할인`)는 유지.
- **최저가 판매처 용량이 기본 용량과 다르면 최저가 우측에 용량 표기**:
  `14,250원 (185매)` → 다른 사이즈 팩임을 드러내 79%의 맥락을 설명.
- 판매처별 단위당가(`매당/g당/ml당`), 판매처 용량(`185매`)도 단위에 맞게 표시.

## 4. 운영자 시트 입력 방법 (구현 후 변경점)

`products` 시트 컬럼 (관련 항목만):

| 컬럼 | 의미 | 입력 예 | 비고 |
|------|------|---------|------|
| `volume_ml` | 용량 **숫자** | `70`, `50`, `185` | 단위 빼고 숫자만 |
| `volume_unit` | 용량 **단위** | `ml` / `g` / `매` | **비우면 ml**. 패드=`매`, 그램 제품=`g` |
| `regular_price` | 정가/MSRP | `₩26,000` / `26000` | 대표 용량 1개 기준. ₩·콤마 OK |

### 입력 예시
| 제품 | volume_ml | volume_unit | regular_price | 화면 표시 |
|------|-----------|-------------|---------------|-----------|
| 선크림 50ml | `50` | (비움) | `₩20,000` | `50ml`, `ml당 …` |
| 시카 패드 70매 | `70` | `매` | `₩26,000` | `70매`, `매당 …` |
| 밤 타입 15g | `15` | `g` | `₩33,000` | `15g`, `g당 …` |

### 입력 시 주의
- **정가는 대표 용량(volume_ml) 기준 1팩 가격**으로 적는다. (대용량 팩 가격을 적으면 안 됨)
- 단위는 `ml`/`g`/`매` 중 하나. `장`·`시트`·`p`는 자동으로 `매` 처리.
- 정가가 비면 할인이 안 보인다(의도된 동작). 할인 노출을 원하면 정가를 채운다.

## 5. 데이터/코드 변경 요약 (참고)

### Part A — 단위 인프라
- **DB:** 마이그레이션 `0017` — `products.volume_unit text not null default 'ml'` (additive).
- **시트 import:** 헤더에 `volume_unit` 추가, 검증(정규화/폴백), 저장.
- **타입/쿼리:** `Product.volume_unit`, `UIProduct.{volumeUnit, volumeMl}`,
  `UIStorePrice.volumeUnit`, 용량 문자열을 단위 반영.
- **포맷:** `perMl()` → `perUnit(price, unit)` 로 일반화 (`ml당/매당/g당`).

### Part B — 표시 정리
- 판매처 카드/표, 홈 픽 카드, 상품 상세의 용량·단위당가·정가 표시를 위 3장 규칙대로 수정.

> **할인율 계산(2-3) 로직은 변경 없음.** 단위는 표시·라벨에만 영향.
> 판매처별 단위 교차검증(listing이 제품 단위와 다르게 파싱될 때 경고)은 **이번 범위 제외**(후속).

## 6. 롤아웃 절차 (구현/머지 후)

1. 마이그레이션 `0017` 적용 (additive — products는 `select *`라 뷰/grant 변경 불필요)
2. `sheets:headers` — 시트에 `volume_unit` 헤더 동기화
3. **운영자:** 패드=`매`, 그램 제품=`g` 로 `volume_unit` 입력 (ml 제품은 비워도 됨)
4. `crawler:sync` — 재import (= `sheets:import` 포함)로 단위/정가 DB 반영
5. `cf:deploy` — 웹 배포

## 7. 엣지 케이스 / 알아둘 점

- **정가 형식:** `₩`·콤마·`원` 포함 입력 가능(숫자만 추출). 0/음수/빈칸 → 정가 없음.
- **용량 다른 팩:** 최저가가 대표 용량과 다른 사이즈(예: 185매)면 단위당가 기준으로
  할인율이 커질 수 있다 → 상세에서 최저가 옆 용량 표기로 맥락 제공.
- **정가 미입력 제품:** 할인 배지·정가·픽 노출 없음. (잘못된 할인 표시보다 미표시가 안전)
- **ml 제품:** `volume_unit`을 비워도 기본 `ml` → 기존과 동일하게 동작.
